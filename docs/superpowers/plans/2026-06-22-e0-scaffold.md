# PAS E0 Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the PAS pnpm monorepo foundation with a runnable NestJS API, Next.js 15 web app, shared domain types, Prisma schema, local PostgreSQL/Redis/MinIO compose stack, client abstractions, tests, and CI.

**Architecture:** The root workspace coordinates three packages: `api`, `web`, and `@pas/shared`. The API validates all required environment variables at startup, depends on external capabilities only through NestJS injection tokens, and uses `RagflowClientMock` for the E0 demo path. PostgreSQL owns structured PAS data while RAGFlow identifiers are stored only as metadata references.

**Tech Stack:** pnpm workspace, Node.js 22+, TypeScript strict, NestJS 11, Next.js 15, Prisma 6, PostgreSQL 16, Redis 7, MinIO, Zod, Vitest, ESLint, Docker Compose, GitHub Actions.
