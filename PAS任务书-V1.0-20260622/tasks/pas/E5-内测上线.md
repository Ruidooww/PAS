# E5 内测上线 — 详细 spec

> 配套 [PAS-任务清单-V1.0.md](../../PAS-任务清单-V1.0.md) ｜ 里程碑 M4
> 依赖：E2、E3、E4 全部完成

---

## 1. 模块定位

把 PAS 从"开发机能跑"推到"5-10 个同事日常能用"。端到端验证 + 生产部署 + 反馈闭环。

## 2. 功能规格

### 2.1 端到端 smoke（E5.1）

```
自动化覆盖三条价值线：
  1. 登录 → 问答 → 出带出处答案
  2. 从第三方 CRM 读客户/商机 → 选客户 → 生成方案 → 导出
     （客户/商机为只读，不在 PAS 创建；mock crmClient 跑 CI）
  3. 外部 token 打 internal 路由 → 403（安全回归）
```

- 进 CI，每次合并前跑

### 2.2 生产部署（E5.2）

```
docker-compose.prod.yml：
  pas-api + pas-web + pas-worker(BullMQ)
  + postgres + redis + minio
  + RAGFlow 全栈（独立 compose，网络互通）
部署 SOP：docs/sop/pas-deploy.md
```

- 部署目标：公司物理机 / 云 VM（非开发机）
- 数据卷持久化 + 定期 pg_dump 备份
- 环境变量从 `.env.prod`（密码强随机，不用默认）

### 2.3 反馈看板（E5.3）

- 聚合 `ConversationFeedback`：使用量 / 👍👎率 / 拒答率
- 简易页面（admin 可见）
- 为 v2 知识缺口分析铺路

## 3. 验收标准（Launch DoD）

### Hard（必过）

- [ ] 内网员工浏览器登录（飞书 OAuth）
- [ ] 50 个真实 query 拒答率 ≤ 10%
- [ ] 多轮对话 3 轮上下文正确
- [ ] Docker 重启 PG/RAGFlow 数据不丢
- [ ] 端到端响应 ≤ 30s
- [ ] 外部用户无法触达任何内部资源（安全）

### Soft（尽量）

- [ ] 答案带截图比例 ≥ 40%
- [ ] 5 人各跑 20 query，反馈率 ≥ 50%
- [ ] 错误日志 < 5 次/天

## 4. 风险

| 风险 | 缓解 |
|------|------|
| IdP 接入慢 | 兜底：本地账号预创建先内测 |
| 拒答率高 | RAGFlow 检索调优 + 知识库补文档 |
| 服务器没到位 | 兜底：内网 IP + Docker 临时跑 |
| 方案生成慢/贵 | 限 history 长度 + 队列限流 |

## 5. 不在本模块（后置 v2）

- 多渠道接入（公众号/客服）
- 用量/成本监控
- 知识缺口 admin 看板
- 经营驾驶舱
