# E4 客户/商机管理 — 详细 spec

> 配套 [PAS-任务清单-V1.0.md](../../PAS-任务清单-V1.0.md) ｜ 里程碑 M3
> 依赖：E0、E1
> **CRM 策略（2026-06-22 定）**：阶段一接**现有第三方 CRM**（API 只读为主），阶段二（v2/v3）自研替换。
> 业务层只认 `crmClient` 接口，换数据源 = 换适配器，业务代码不动（D2）。具体接哪家 CRM 与本 spec 无关。

---

## 1. 模块定位

PAS 的客户/商机数据接入层。**MVP 不自建 CRM**——客户/商机主数据在第三方 CRM，PAS 通过 `crmClient` 适配器**读**，自己只存"PAS 原生扩展"（方案关联、知识标签等）+ 客户引用。这样将来自研 CRM 时只换适配器、迁移数据，业务层零改动。

## 2. 用户故事

- 作为**售前**，我在 PAS 里看到的客户/商机来自现有 CRM，不用重复录入
- 作为**售前**，我从客户/商机详情发起方案生成，自动带入客户信息
- 作为**架构**，将来 PAS 自研 CRM 上线，业务代码不用改，只换 `crmClient` 实现

## 3. 功能规格

### 3.1 crmClient 抽象（E4.1）★ 核心

```typescript
// clients/crm.ts —— 接口按 PAS 自己的领域模型设计，不让任何 CRM 厂家字段结构泄漏
interface CrmClient {
  getCustomer(ref: CustomerRef): Promise<Customer>
  listCustomers(p: { q?; ownerId?; page? }): Promise<Customer[]>
  listOpportunities(p: { customerRef?; stage?; page? }): Promise<Opportunity[]>
  // 阶段一：写操作尽量少（回写第三方脆弱）；如需建商机，谨慎封装
}

// 实现：
//   ExternalCrmClient —— 阶段一，包现有第三方 CRM 的 API（只读为主）
//   PasCrmClient      —— 阶段二，读 PAS 自己的 Postgres
// 注入哪个由 config 决定（CRM_PROVIDER=external|pas）
```

- **领域类型 PAS 自定义**：`Customer{ ref, name, industry, scale, ownerId }`，把第三方返回的数据**适配进**这个类型
- 阶段一 `ExternalCrmClient` 调第三方 API；阶段二 `PasCrmClient` 读本地表——接口不变
- 第三方厂家细节（端点/认证/字段映射）全封死在 `ExternalCrmClient` 内部，对外不可见

### 3.2 客户/商机读取（E4.2）

```
GET /api/internal/customers?q=&page=   → crmClient.listCustomers（背后第三方）
GET /api/internal/customers/:ref       → crmClient.getCustomer
GET /api/internal/opportunities?...    → crmClient.listOpportunities
```

- **阶段一只读第三方**，PAS 不提供客户/商机的增删改（在第三方系统里维护）
- 性能：第三方 API 通常有限流 → PAS 侧加缓存（Redis）+ 可选轻量镜像表（见 §4）
- 商机阶段、金额等字段从第三方映射到 PAS 领域类型

### 3.3 PAS 原生扩展 + 关联（E4.3）★

```
PAS 自己存的（不在第三方 CRM）：
  - Proposal ──→ customerRef（指向第三方客户 id）
  - 客户的知识标签 / 方案历史 / PAS 内部备注

关联查询：
  GET /customers/:ref → crmClient 取客户基本信息
                      + PAS 库取该客户的方案列表
  从客户/商机详情"生成方案" → 带 customerRef 进 E3
```

- **方案必须挂客户**，但挂的是 `customerRef`（可外部 id，可本地 FK）——阶段二迁移友好

### 3.4 前端（E4.4）

- 客户列表（搜索/筛选，数据来自第三方 CRM）+ 详情页（含 PAS 方案列表）
- 商机列表 + 详情
- 详情页"生成方案"按钮 → 跳 E3 带 customerRef

## 4. 数据模型（PAS 侧）

```prisma
// 阶段一：PAS 不持有客户主数据，只存引用 + 原生扩展
// 可选：轻量镜像表缓存第三方客户（减少 API 调用 + 支持本地关联查询）
model CustomerMirror {
  ref         String   @id              // 第三方客户 id（阶段二可迁为本地 cuid）
  source      String   @default("external")  // external|pas —— 阶段二改 pas
  name        String
  industry    String?
  scale       Int?
  ownerId     String?
  syncedAt    DateTime                  // 最近同步时间
  proposals   Proposal[]
  @@index([ownerId])
}
// Proposal.customerId 改为 customerRef String，关联 CustomerMirror.ref
```

> `source` + `ref` 设计是阶段二迁移的关键：自研后把 source 改 `pas`、ref 迁本地 id，关联不断。

## 5. 验收标准（DoD）

- [ ] `crmClient` 接口定义清晰，`ExternalCrmClient` 实现接通现有 CRM（能拉到真实客户/商机）
- [ ] PAS 客户列表/详情展示第三方数据
- [ ] 方案能挂 `customerRef` 并在客户详情回显
- [ ] 第三方 API 限流有缓存兜底，不因限流报错
- [ ] `amountEst` 等敏感字段对外部用户脱敏（E1.6 联动）
- [ ] 切 `CRM_PROVIDER=pas`（mock 实现）业务层不报错 —— 证明抽象有效

## 6. 关键前置验证（决定阶段二可行性）

⚠ **趁现在确认第三方 CRM 数据可携性**（替换的命门）：
- 开放 API 能否读全客户/联系人/商机/合同
- 能否批量导出历史数据（阶段二迁移要全量搬出）
- 限流/字段/条数限制

第三方锁数据 → 阶段二迁移会痛，需提前知道。

## 7. 不在本模块（后置 v2）

- **自研 CRM 替换第三方**（阶段二：换 `PasCrmClient` + 数据迁移）
- 商机自动抓取（机器人/语音/招投标爬虫）
- 客户画像自动构建（M15 客情系统 CIP）
- 联系人/决策链关系图
- 向第三方 CRM 大量回写（阶段一只读为主）
