## 1. 当前共识

PAS 的初始路线图是完整产品蓝图，包含 M1~M15 等模块。任务书 V1.0 是从路线图中裁剪出来的第一阶段目标：

> 先做一个能跑起来、能内测、能支撑最小售前闭环的 PAS。

因此，路线图与任务书的关系是：

```text
路线图：定义 PAS 最终要长成什么样
任务书 V1.0：定义第一阶段先让 PAS 动起来
仓库代码：任务书落地后的实现结果
```

当前不应该把路线图里的所有能力都塞进 V1.0。路线图中的完整知识库、完整方案生成引擎、Agent 编排、驾驶舱、多渠道、渠道管理等能力，应该按 V2 / V3 / V4 逐步实现。

## 3. V1.0 的正确定位

V1.0 不等于完整 PAS 系统，而是：

```text
客户输入
  ↓
需求结构化 / 简化处理
  ↓
方案生成
  ↓
输出可展示 / 可下载结果
```

V1.0 的核心目标是：

- 能运行
- 能演示
- 能内测
- 能跑通售前最小闭环
- 不追求完整 AI 平台能力

V1.0 不应包含：

- 完整 RAG pipeline
- 完整知识图谱
- 完整 Agent Runtime
- Workflow Engine
- Plugin Marketplace
- 多租户平台化
- 驾驶舱 / 渠道管理 / 多渠道完整系统

## 4. V1 → V4 演进边界

### V1：MVP 闭环系统

目标：先让 PAS 能用起来。

允许：

- M1 售前主链路
- M15 客情输入的简化版本
- M9 权限的简化 RBAC
- Mock / 简化 AI 能力
- 基础 API / Web / 数据模型 / 部署能力

禁止：

- 完整 RAG
- 完整知识库 pipeline
- 知识图谱
- Agent 编排
- Workflow Engine
- 插件市场

---

### V2：插件化 AI 能力

目标：把 AI 能力从写死实现变为可替换插件。

允许：

- LLM Plugin
- RAG Plugin
- RAGFlow Plugin
- FastGPT Plugin
- KB Plugin
- Embedding Plugin
- Config-based provider switch

禁止：

- Agent Planner
- Agent Executor
- Multi-Agent
- Workflow DAG

V2 的核心变化：

```text
Service → PluginManager → Plugins
```

---

### V3：Agent 系统

目标：让系统具备任务拆解与工具调用能力。

允许：

- AgentRuntime
- Planner
- Executor
- Tool Registry
- Task Memory
- Tool Calling

禁止：

- 多租户平台
- 插件市场
- 多 Agent 商业化编排

V3 的核心变化：

```text
Service → AgentRuntime → Planner → Executor → Tools
```

---

### V4：平台化系统

目标：从单一 AI 系统升级为可扩展 AI 平台。

允许：

- Multi-Agent Manager
- Workflow Engine
- Plugin Registry / Plugin Marketplace
- Tenant Isolation
- Task Scheduler
- SaaS 化能力

V4 的核心变化：

```text
Single Agent System → Multi-Agent + Workflow + Plugin Ecosystem + Multi-Tenant Platform
```

## 5. 插件化架构设计原则

PAS 不应该直接绑定 RAGFlow、FastGPT、OpenAI 或任何具体 provider。正确设计是：

```text
Service
  ↓
抽象接口 / PluginManager
  ↓
具体插件实现：RAGFlowPlugin / FastGPTPlugin / OpenAIPlugin / MockPlugin
```

### 错误方式

```text
Service → 直接调用 RagflowClient
Service → 直接调用 FastGPT
Service → 直接调用 OpenAI
```

问题：

- 业务层绑定 provider
- 后续切换 provider 需要改业务代码
- V2/V3 升级成本变高
- 系统变成“RAGFlow 系统”或“FastGPT 系统”，而不是 PAS 系统

### 正确方式

```text
Service → RAG Plugin Interface → RagflowPlugin / FastGPTPlugin / MockRagPlugin
```

RAGFlow / FastGPT 的定位：

> 它们只是 RAG 能力的 provider/plugin 实现，不是 PAS 的核心架构。

## 6. 插件注册的含义

这里说的“插件注册”不是去某个网站注册，也不是 GitHub 插件市场注册。

它只是 PAS 本地代码中的运行时能力注册：

```ts
pluginManager.register(new MockLLMPlugin())
pluginManager.register(new RagflowPlugin())
pluginManager.register(new FastGPTPlugin())
```

也可以通过配置驱动：

```yaml
plugins:
  llm: openai
  rag: ragflow
  kb: mock
  embedding: mock
```

推荐阶段：

- V1 / V1.5：使用 config-based 注册，简单可控。
- V2 以后：逐步支持自动发现 / decorator 注册。

## 7. Service 层纪律

Service 层只能做编排，不允许写具体 AI/provider 逻辑。

### Service 层禁止

- 直接 import RagflowClient
- 直接 import FastGPTClient
- 直接调用 OpenAI
- 直接调用 Embedding API
- 直接访问具体 vector DB provider
- 写 provider-specific 参数
- 写 Agent planning 逻辑

### Service 层允许

- 调用抽象 interface
- 调用 PluginManager
- 调用 facade service
- 拼装业务输入输出

正确示例：

```ts
const rag = pluginManager.get("rag")
const llm = pluginManager.get("llm")

const context = await rag.execute(input)
const result = await llm.execute({ input, context })

return result
```

## 9. Codex 工业级执行系统

为了避免 Codex 后续执行时越级、混层、乱改，建议引入三层治理：

```text
Issue
  ↓
Phase 判断
  ↓
Codex Prompt
  ↓
执行范围
  ↓
Guard 检查
  ↓
PR 模板验收
  ↓
CI 拦截越级
```

### 9.1 文档治理层

建议新增：

```text
docs/execution/current-phase.md
docs/execution/phase-boundaries.md
docs/execution/codex-rules.md
.codex/prompts/base-executor.md
.codex/prompts/v2-plugin-task.md
```

### 9.2 GitHub 模板层

建议新增：

```text
.github/ISSUE_TEMPLATE/pas-task.yml
.github/ISSUE_TEMPLATE/pas-decision.yml
.github/pull_request_template.md
```

### 9.3 自动 Guard 层

建议新增：

```text
tools/guards/check-direct-provider-imports.ts
tools/guards/check-service-boundary.ts
tools/guards/check-forbidden-keywords.ts
```

并在根 `package.json` 增加：

```json
{
  "scripts": {
    "guard": "pnpm guard:imports && pnpm guard:service && pnpm guard:phase",
    "guard:imports": "tsx tools/guards/check-direct-provider-imports.ts",
    "guard:service": "tsx tools/guards/check-service-boundary.ts",
    "guard:phase": "tsx tools/guards/check-forbidden-keywords.ts"
  }
}
```
